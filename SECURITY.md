# Security Policy

## Reporting a Vulnerability

Although this is a documentation project and does not contain executable code, we take security seriously.

### What to Report

- Malicious links or URLs in the documentation
- External resources that have been compromised
- Suggestions for unsafe practices in code examples
- Any content that could lead users to security risks

### How to Report

**Please do NOT report security issues through public GitHub issues.**

Instead, report vulnerabilities by:

1. **Email:** Send details to the repository owner via GitHub profile
2. **Private Vulnerability Reporting:** Use [GitHub's private vulnerability reporting](https://github.com/toxicbishop/markdown-guide/security/advisories/new) feature

### What to Include

- Description of the vulnerability
- Location in the repository (file, line number)
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial Assessment:** Within 7 days
- **Resolution:** Depends on severity, typically within 14 days

### Scope

This security policy applies to:

- All Markdown files in this repository
- External links and resources referenced
- Code examples provided in documentation

### Out of Scope

- Issues with GitHub's platform itself
- Third-party services linked in badges
- User-generated content in issues/discussions

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x.x   | Yes       |

## Recognition

We appreciate responsible disclosure. Contributors who report valid security issues will be acknowledged in our CHANGELOG (unless they prefer to remain anonymous).

Thank you for helping keep this project safe.
