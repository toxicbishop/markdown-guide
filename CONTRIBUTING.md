# Contributing to Markdown Guide

Thank you for your interest in contributing to this project. This guide will help you get started.

## How to Contribute

### 1. Fork the Repository

Click the "Fork" button at the top right of this repository.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/markdown-guide.git
cd markdown-guide
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

Use descriptive branch names:
- `feature/add-new-section` for new content
- `fix/typo-in-tables` for corrections
- `docs/improve-examples` for documentation improvements

### 4. Make Your Changes

- Follow the existing formatting style
- Test that all Markdown renders correctly on GitHub
- Keep examples clear and concise

### 5. Commit Your Changes

```bash
git add .
git commit -m "Add: description of your changes"
```

Commit message prefixes:
- `Add:` for new content
- `Fix:` for corrections
- `Update:` for improvements to existing content
- `Remove:` for deletions

### 6. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 7. Open a Pull Request

Go to the original repository and click "New Pull Request". Select your branch and provide a clear description of your changes.

## Contribution Guidelines

### Content Standards

- Use proper Markdown syntax
- Include both the rendered example and the source code
- Keep explanations concise and practical
- Test all examples on GitHub before submitting

### What We Accept

- New Markdown features or techniques
- Corrections to existing content
- Better examples or explanations
- Translations (in separate files)

### What We Don't Accept

- Content unrelated to Markdown
- Duplicate sections
- Promotional or advertising content
- Changes that break existing formatting

## Code of Conduct

- Be respectful and constructive
- Welcome newcomers
- Focus on collaboration

## Questions?

Open an issue if you have questions or need clarification before contributing.
